# Документация API для parse_form.py

## Общее описание
Данный модуль представляет собой FastAPI-приложение для обработки лекций: загрузки планов, аудиозаписей, транскрибации и сравнения с помощью GigaChat API.

## Вспомогательные функции

### `save_file(destination_dir: str, file_type: str, file: UploadFile) -> str`
Сохраняет загруженный файл в указанную директорию.

**Параметры:**
- `destination_dir` - путь к директории для сохранения
- `file_type` - тип файла (для формирования имени)
- `file` - загружаемый файл

**Возвращает:** путь к сохраненному файлу

---

## Эндпоинты (Endpoints)

### Главная страница
#### `GET /`
Отображает главную страницу приложения.

**Возвращает:** HTML-страницу `start_window.html`

---

### Шаг 1: Выбор способа загрузки плана
#### `GET /step1`
Отображает страницу выбора способа загрузки плана лекции.

**Возвращает:** HTML-страницу `step_one.html`

---

### Шаг 2: Загрузка аудиофайла
#### `GET /step2`
Отображает страницу загрузки аудиофайла лекции.

**Возвращает:** HTML-страницу `step_two.html`

---

### Шаг 3: Сравнение лекции и плана
#### `GET /step3`
Отображает страницу с результатами сравнения лекции и её плана.

**Возвращает:** HTML-страницу `step_three.html` или JSON с ошибкой

---

### Загрузка плана лекции
#### `GET /upload-plan`
Отображает страницу загрузки файла плана лекции.

**Возвращает:** HTML-страницу `upload_plan.html` или JSON с ошибкой

---

### Форма для ввода плана
#### `GET /lectures`
Отображает форму для ручного ввода структуры лекции.

**Возвращает:** HTML-страницу `form.html` или JSON с ошибкой

---

### Загрузка готового транскрипта
#### `GET /upload-transcript`
Отображает страницу загрузки готового текста транскрипта.

**Возвращает:** HTML-страницу `upload_audio.html` или JSON с ошибкой

---

### Сохранение загруженного плана
#### `POST /save-plan`
Сохраняет загруженный файл плана лекции.

**Параметры:**
- `file` - загружаемый файл (multipart/form-data)

**Возвращает:** 
- Успех: редирект на `/plan-to-json`
- Ошибка: JSON с статусом "fail"

---

### Конвертация плана в JSON
#### `POST /plan-to-json`
Конвертирует загруженный файл плана в JSON-формат.

**Возвращает:** JSON с информацией о результате операции:
```json
{
    "status": "success" | "fail",
    "redirect_url": "/step2" | null,
    "error_msg": null | string
}
```

### Сохранение аудиофайла
#### POST /save-audiofile

Сохраняет загруженный аудиофайл лекции.

**Параметры:** file - загружаемый аудиофайл (multipart/form-data)

**Возвращает:** JSON с информацией о результате операции:
```json

{
    "status": "success" | "fail",
    "message": string,
    "error_msg": null | string
}
```

### Транскрибация аудио
#### POST /speech-transcribe-request

Запускает процесс транскрибации загруженного аудиофайла.

**Возвращает:** JSON с информацией о результате операции:
```json

{
    "message": "success" | "fail",
    "redirect_url": "/step3" | null,
    "error_msg": null | string
}
```

### Запрос к GigaChat API
#### POST /gigachat-api-request

Отправляет запрос к GigaChat API для сравнения плана и транскрипта лекции.

**Возвращает:** JSON с информацией о результате операции:
```json

{
    "message": "success" | "fail",
    "gigachat_response_saved_to": string | null,
    "error_msg": null | string
}
```

### Сохранение загруженного транскрипта
#### POST /save-transcript

Сохраняет загруженный файл транскрипта.

**Параметры:** file - загружаемый файл транскрипта (multipart/form-data)

**Возвращает:** JSON с информацией о результате операции:
```json

{
    "status": "success" | "fail",
    "message": string,
    "redirect_url": "/step3" | null,
    "error_msg": null | string
}
```

### Отображение анализа
#### POST /show-in-browser

Загружает и возвращает результат анализа для отображения в браузере.

**Возвращает:** JSON с содержимым анализа:
```json

{
    "status": "success" | "fail",
    "gigachat_answer": string | null,
    "error_msg": null | string
}
```

### Скачивание файла анализа
#### GET /download

Скачивает файл с результатами анализа в формате JSON.

**Возвращает:** FileResponse с файлом lecture_analysis.md

## Глобальные константы

|Константа|Описание|    
|:---|:---|
|LECTURES_JSONS_DIR|директория для хранения JSON планов|
|TRANSCRIPTS_DIR|директория для хранения транскриптов|
|UPLOADED_AUDIOFILES_DIR|директория для временного хранения аудиофайлов|
|UPLOADED_PLANS_DIR|директория для временного хранения планов|
|COMPARISON_JSONS_DIR|директория для хранения результатов сравнения|
|FILES_TO_PROCESS|словарь для хранения путей к обрабатываемым файлам|
